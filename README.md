## 📌 Описание проекта
Данный проект посвящен анализу демографической трансформации регионов России. На основе данных Росстата исследованы векторы перемещения населения, выявлены скрытые процессы субурбанизации и оценен реальный масштаб «стягивания» человеческих ресурсов в крупнейшие агломерации.

**Ключевой вопрос:** Куда на самом деле едут люди и как меняется структура расселения в постиндустриальной экономике?

## 🔍 Основные гипотезы и результаты

| Гипотеза | Статус | Ключевой инсайт |
| :--- | :---: | :--- |
| **1. Столичный пылесос** | ✅ Подтверждена | Соседи Москвы и СПб теряют население в 2.5 раза быстрее, чем в среднем по стране. Близость к столицам — фактор риска, а не роста. |
| **2. Ложная субурбанизация** | ✅ Подтверждена | В Тюмени, Калининграде и на Юге рост села — это расширение городов. Село становится «спальным районом» без аграрной функции. |

## 🛠 Технологический стек и методы
Проект реализован в виде полного ETL-пайплайна:

1.  **Data Extraction:** Автоматизированный парсинг иерархических Excel-отчетов Росстата (обработка вложенных категорий через `ffill`).
2.  **Data Cleaning:** * Нормализация кодов ОКАТО/ОКТМО.
    * Фильтрация агрегатов федерального уровня.
    * Обработка выбросов (Outliers) методом **Z-score** на логарифмированных данных.
3.  **Feature Engineering:** Расчет коэффициентов интенсивности миграции на 1000 жителей (промилле).
4.  **Imputation:** Восстановление временных рядов методом **линейной интерполяции**.
5.  **Visualization:** Построение корреляционных диаграмм (Scatter Plots) и расходящихся столбчатых диаграмм (Diverging Bars) для выявления структурных сдвигов.

## 📁 Архитектура проекта

### Логика обработки данных (ETL Pipeline)
Проект организован по принципу слоистого хранения данных, что обеспечивает воспроизводимость анализа:
1. **Raw Layer** (`data/raw/`): Неизменяемые исходные Excel-файлы Росстата.
2. **Primary Layer** (`data/primary/`): Результат парсинга и нормализации (CSV).
3. **Clean Layer** (`data/input_clean/`): Данные после восстановления пропусков и фильтрации выбросов.
4. **Report Layer** (`data/reports/`): Агрегированные показатели и результаты классификации для визуализации.

### Описание файлов кода
* `refactor_data.ipynb` — Конвертация Raw -> Primary (очистка структуры Excel).
* `convert_data.ipynb` — Трансформация Primary -> Clean (интерполяция, Z-score).
* `summary.ipynb` — Аналитика и визуализация на базе Report-слоя.
